# Key points extracted during ingest

- **The Problem of Slop**: Low-quality, AI-generated code is flooding repositories, often generated by "clankers" (poorly performing agents).
- **Compounding Booboos**: Small, unchecked AI errors accumulate rapidly, leading to systems that are difficult to understand or maintain.
- **Pi Philosophy**: A minimal, extensible coding agent harness designed for observability, model choice, and malleability.
- **Malleable Agents**: The core idea that an agent should be modifiable by both the human and the agent itself to fit specific workflows.
- **Minimal Tooling**: Pi uses only four core tools (read, write, edit, bash) to reduce token overhead and keep the system prompt small.
- **Friction as a Feature**: Humans should stay "in the code" to build understanding and taste, rather than outsourcing everything to agents.
- **Terminal-Bench 2.0**: A benchmark showing that minimal harnesses often outperform complex ones across different model families.
- **Pi Components**:
    - `pi-ai`: Unified LLM API.
    - `pi-agent-core`: Core agent loop.
    - `pi-tui`: Bespoke terminal UI framework.
    - `pi-coding-agent`: The CLI providing session management and project context.
