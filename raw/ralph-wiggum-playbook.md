# Key points extracted during ingest

- Blog post summarizing the Ralph Wiggum methodology from Huntley/Farr
- Three core principles: context is scarce, plans are disposable, backpressure beats direction
- "Human roles shift from telling the agent what to do to engineering conditions where good outcomes emerge naturally through iteration" — Clayton Farr
- Dumb zone hits around 40% context utilization; one-task-per-iteration sidesteps this
- Three backpressure layers: downstream gates (tests/lint), upstream steering (code patterns), LLM-as-judge (subjective criteria)
- Plan disposability: treat IMPLEMENTATION_PLAN.md as coordination state, not a contract
- Topic scope test: describe in one sentence without "and" — if you need conjunctions, split
- Limitations: bad specs (garbage in), architectural decisions (still need human), cost ($50-100+ for 50 iterations on large codebase)
