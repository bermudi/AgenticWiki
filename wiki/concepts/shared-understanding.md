# Shared Understanding

> The collective mental model that lets a team communicate, reason, and generate ideas together; built through shared spaces where humans and agents can co-discuss work.

## The Concept

[[geoffrey-litt|Geoffrey Litt]] shifts the AI-coding conversation from solo understanding to team understanding. The real constraint, he argues, is often that a whole team needs to understand the system before they can jam on it. Shared understanding is the substrate of effective collaboration: shared names for parts of the system, shared mental models, shared vocabulary.

## Shared Spaces as the Mechanism

Litt's proposed mechanism is the **shared space** — a collaborative document or chat where humans and agents see each other's communications. The comparison is moving from one-on-one conversations to a Slack channel: more behavior is visible, so the team understands together.

Examples from the talk:

- Multiplayer chat threads with multiple humans and agents together.
- Collaborative documents (e.g., in Notion) where a plan generated by an agent can be commented on and discussed by the team.
- Coding agents embedded in the shared workspace, so code and discussion live in the same place.

## Relation to Existing Ideas

Shared understanding is the team-level counterpart to the [[shared-design-concept]] — the "theory of the code" that both human and agent must share to produce coherent work. The shared-design-concept is about the code theory; shared understanding is about the team's collective grasp of that theory.
