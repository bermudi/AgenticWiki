# Lavish

> An HTML artifact editor for planning with an agent. The agent renders design options as a single-page web UI in the project's own design system; the human annotates, selects, and decides in the browser instead of reading a wall of text in the terminal.

## How It Works

When asked to plan a feature, the agent produces options in an HTML artifact generated by Lavish. The artifact:

- Uses the **same design system as the project being worked on**, so the mockups look like the real app.
- Presents options visually, making trade-offs easier to compare than a terminal dump.
- Allows the human to **annotate specific parts** and provide feedback inline.
- Includes decision buttons or checkboxes so the human can make choices without returning to the terminal.

Once the human is happy, they instruct the agent to start building, and the agent proceeds to implementation with the requirements already clarified.

## Why It Matters

Kun Chen's claim is that the default plan mode produces a dense wall of text that is hard to parse and hard to give precise feedback on. Lavish moves the plan into a visual medium that the human can scan and annotate, reducing the friction of the planning phase and making the resulting requirements clearer.
